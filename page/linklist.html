Linked List Data Structure

A Linked List is a linear data structure in which elements (nodes) are connected using pointers. Each node consists of two parts: the data and a reference to the next node in the sequence. Unlike arrays, linked lists are dynamic in size and can efficiently handle insertions and deletions without requiring shifting of elements. Linked Lists are categorized into different types, such as singly linked lists, doubly linked lists, and circular linked lists, each with unique properties and use cases.

Time Complexity:

- Access by Index: O(n) (linear time, as traversal from the head is required).
- Insertion at Head: O(1) (constant time to update the head pointer).
- Insertion at Tail: O(1) (constant time if tail pointer is maintained, otherwise O(n)).
- Deletion: O(1) for head deletion, O(n) for arbitrary node deletion (traversal needed).
- Search: O(n) (linear time to traverse the list for the desired value).

Space Complexity:

- The space complexity of a linked list is O(n), where n is the number of nodes in the list. Each node requires additional space for the pointer/reference to the next node. - In the case of a doubly linked list, space complexity increases as each node stores an additional pointer (to the previous node), resulting in O(2n) = O(n).

Linked lists are widely used in scenarios requiring efficient memory utilization, dynamic data handling, and operations like insertion and deletion. Applications include implementation of stacks, queues, and adjacency lists for graphs.
