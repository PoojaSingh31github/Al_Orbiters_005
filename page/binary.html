Binary Search Algorithm

Binary Search is an efficient algorithm for finding a target element in a sorted array. It works by repeatedly dividing the search interval in half. If the target value matches the middle element, the search is complete. Otherwise, the search continues in the left or right half, depending on whether the target value is smaller or larger than the middle element. Binary Search reduces the problem size by half at each step, making it significantly faster than linear search for large datasets.

Time Complexity:

- Best Case: O(1) (when the target is the middle element).
- Worst Case: O(log n) (in every step, the array size reduces by half).
- Average Case: O(log n) (applies when the target could be anywhere in the sorted array).

Space Complexity:

- Iterative Implementation: O(1) (constant space as it does not require additional data structures).
- Recursive Implementation: O(log n) (due to the recursive stack used for function calls).

How Binary Search Works:

1. Start with two pointers: low at the beginning and high at the end of the array. 2. Calculate the middle index: mid = (low + high) // 2. 3. Compare the middle element with the target value: - If array[mid] == target, the target is found. - If array[mid] > target, narrow the search to the left half: high = mid - 1. - If array[mid] < target, narrow the search to the right half: low = mid + 1. 4. Repeat steps 2-3 until low > high or the target is found.

Binary Search is widely used in scenarios requiring efficient search operations, such as database indexing, sorted arrays, and search engines. Its logarithmic time complexity makes it a critical tool for handling large datasets efficiently.
